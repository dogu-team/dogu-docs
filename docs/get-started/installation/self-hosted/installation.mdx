This document describes how to get started with Dogu in a Self-Hosted environment.

## Hardware Requirements

|                          | Minimum Requirements                                 |
| :----------------------: | ---------------------------------------------------- |
| <b>Operating System</b>  | Ubuntu(x86_64), macOS(x86_64,arm64), Windows(x86_64) |
|       <b> CPU </b>       | 4 Cores                                              |
|     <b> Memory </b>      | 8 GB RAM                                             |
| <b> Free Disk Space </b> | SSD 128GB                                            |
|  <b> Dependencies </b>   | Docker, Docker-Compose                               |

## Installing Dogu with Docker

### Preparing Docker and Docker-compose

If Docker and Docker-compose are not installed, please refer to the links below for installation.

- [Install Docker](https://docs.docker.com/engine/install/)
- [Install Docker-compose](https://docs.docker.com/compose/install/)

### Creating Mount Paths

<Tabs groupId="os" defaultValue="linux" queryString>

<TabItem value="linux" label="Linux(x86_64)" default>

```
sudo mkdir /var/lib/dogu
sudo mkdir /var/lib/dogu/nexus-data
sudo chown 200 -R /var/lib/dogu/nexus-data
```

:::note
The `/var/lib/dogu` path created above can be used as the value for `VOLUME_PATH` in the docker-compose.env file to be created in the next step. This can be ignored for macOS.
:::

</TabItem>

<TabItem value="windows/macOS" label="Windows/macOS">

```
No need to create mount paths for Windows / macOS.
```

</TabItem>
</Tabs>

### Checking out the Repository
Use the Git CLI to check out the dogu-self-hosted Repository.

```
git clone https://github.com/dogu-team/dogu-self-hosted.git dogu-self-hosted
```

After checkout, navigate to the **dogu-self-hosted** folder.

```
cd dogu-self-hosted
```

### Deploying Containers

Run the following command from the **dogu-self-hosted** folder.

<Tabs groupId="arch" defaultValue="Linux(x86_64)" queryString>

<TabItem value="Linux(x86_64)" label="Linux(x86_64)" default>

```
docker compose -f ./docker-compose.yml --env-file ./docker-compose.env up  -d
```

</TabItem>

<TabItem value="macOS(arm64)" label="macOS(arm64)">

```
docker compose -f ./docker-compose-macOS-arm64.yml --env-file ./docker-compose.env up  -d
```

</TabItem>
<TabItem value="windows/macOS(x86_64)" label="windows/macOS(x86_64)">

```
docker compose -f ./docker-compose-macOS-windows-x86_64.yml --env-file ./docker-compose.env up  -d
```

</TabItem>
</Tabs>

### Verifying Container Deployment

Run the following command to check if dogu-self-hosted has been successfully deployed.

```
docker ps
```

If the deployment is successful, you will see the following output.

<img src="/img/get-started/docker-ps.png" />

### Accessing the Console Page

Connect to **http://localhost:3001** to verify that Dogu is running correctly. If it's running successfully, you will see the following screen.
